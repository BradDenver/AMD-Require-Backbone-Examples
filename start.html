  • api
  • assets

  • modules

  • vendorAssets
    • css

api

This folder will be the root folder for our RESTful api. The api will detailed later

assets

The css and img sub folders will house your site specific css and image files

common

This folder will house any common js files that are used accross muliple modules

modules

This folder will have a sub folder for each module you create.

  • module1
    • collections
    • models
    • templates
    • views

Each module folder will have collections, models, templates and views sub folders to hold thier corresponding files. Again this will be covered in more detail later.

vendorAssets

This will hold js and css assets provided by external vendors eg. Twitter Bootstrap

css vendor provided css files

img vendor provided img files

libs vendor provided js libraries. If the lib is more than a single file it should be nested in its own folder eg. require.

mixed libraries such as Twitter Boostrap that contain a mix of css/img/js files/folders.

plugins vendor provided javascript plugins

Downloads

Now we have a folder structure its time to grab some libraries to work with

jQuery

jQuery will be our library of choice for working with the DOM. Head to http://jquery.com/download/ and grab the minified version of the latest release. This will be a file such as jquery-1.8.2.min.js. Rename it to jquery.min.js and place this in the vendorAssets/libs/ folder.

Twitter Bootstrap

Next up is Twitter Bootstrap. This will provide us with a css base to work from as well as some handy js widgets. Grab the latest version from http://twitter.github.com/bootstrap/assets/bootstrap.zip and place the unzipped bootstrap folder in vendorAssets/mixed/.

RequireJS

RequireJS will handle our javascript loading and dependency management. We need to grab two files from http://requirejs.org/docs/download.html. First we need the latest minified version of require.js. Second we need the text plugin which has a direct link https://raw.github.com/requirejs/text/latest/text.js. Place require.js and text.js in vendorAssets/libs/require/.

Underscore.js

Underscore.js is a requirement for Backbone.js and contains some great javascript utilites. We will also make use of its template functionality to compile markup templates. Grab the latest production version from http://underscorejs.org/underscore-min.js and place the underscore-min.js file in the vendorAssets/libs/ folder.

* In the future I may look at swapping out Underscore.js in favour of Lo-Dash but for now I'll stick with the default.

Backbone.js

Last but not least is Backbone.js. This is the framework we will use to build our application. Grab the latest production version from http://backbonejs.org/backbone-min.js and place the backbone-min.js file in the vendorAssets/libs/ folder.

Key Points

          

            
baseUrl

            
We set the baseUrl to point to our modules folders as that is where most of our future files will go

            
paths

            
Path mappings are relative to the base url and provide a shorter syntax when defining core dependancies throughout the project such as backbone

            
shim

            
shims allow us to use traditional "browser globals" scripts that do not use define() to declare the dependencies and set a module value such jQuery

          


          
require( ) call

          

          
The require() function takes two arguments.  The second is a callback function.

          
The first is an array of dependencies. jQuery, underscore and backbone all make use of the paths we have defined in the config. MainView will be our first view (we will create it later). These are all AMD modules so have a return value that is passed to the callback. We also list bootstrap as a dependency so its functions are ready to use. As it is a shim rather than an amd module it has no return value to pass to the function

          
The second argument is the callback function that is called once all dependancies are loaded.
            Note that at this point any dependencies of any modules we have listed (eg MainView) will have also been loaded.
